You are to implement a cooperative editing architecture for Archi (ArchiMate tool).

Architecture:
- Client: Archi plugin (Eclipse RCP/EMF) that captures local model + view changes and applies remote ops.
- Server: Quarkus collaboration server is the authority for validation, ordering (revisions), idempotency, and lock leases. WebSocket API for clients.
- Messaging: Kafka distributes accepted operation batches per model.
- Storage: Neo4j stores both (a) materialized current state and (b) an append-only op log with revisions.

Important constraints:
- Treat Archi-specific notation (views, view objects, connections) as OPAQUE JSON blobs (`notationJson`) in messages and Neo4j.
- The server does not interpret `notationJson`; it only validates IDs/relationships and stores/rebroadcasts.

Deliverables:
1) JSON schemas:
   - OperationBatch envelope (with opBatchId, baseRevision, assignedRevisionRange)
   - Op types: Create/Update/Delete Element/Relationship, Set/UnsetProperty,
     Create/Update/Delete View, Create/Delete ViewObject/Connection,
     UpdateViewObjectOpaque, UpdateConnectionOpaque
   - LockEvent and Presence schemas
   - Canonical notationJson contract (View, ViewObject, Connection) treated as opaque
2) Kafka topic naming, keys, partition strategy:
   - Topics: archi.model.<modelId>.ops / locks / presence
   - 1 partition per model ops topic for total order
3) Neo4j schema:
   - Labels/relationships for Model/Element/Relationship/View/ViewObject/Connection plus Commit/Op
   - Constraints and indexes
4) Quarkus server skeleton:
   - WebSocket endpoint: /models/{modelId}/stream
   - Inbound: Join, SubmitOps, AcquireLock, ReleaseLock, Presence
   - Outbound: CheckoutSnapshot/Delta, OpsAccepted, OpsBroadcast, LockEvent, PresenceBroadcast, Error
   - Internal services: ValidationService, RevisionService, LockService, Neo4jRepository, KafkaPublisher, KafkaConsumer, SessionRegistry
   - Pipeline: validate -> lock check -> assign revisions -> write op log -> apply state -> update headRevision -> publish to Kafka -> broadcast
   - Idempotency/dedup: opBatchId
5) Archi plugin skeleton:
   - EMF change capture and mapping to ops
   - NotationSerializer/Deserializer that round-trips view/viewobject/connection notation to/from JSON maps using the canonical contract
   - Debounced sending of UpdateViewObjectOpaque / UpdateConnectionOpaque
   - Remote op applier with echo suppression
   - Lock acquisition hooks on drag/resize (lease TTL)

Mapping guidance (plugin-side):
- Use com.archimatetool.model EMF interfaces:
  - IDiagramModelObject bounds/fill/alpha/line/font/text alignment
  - IDiagramModelArchimateObject: archimate element ref, type, image info, text position, children
  - IDiagramModelConnection: source/target, bendpoints, type bitmask, text position, name visible
  - IDiagramModelArchimateConnection: relationship ref
- Opaque blobs are full replacements; server remains agnostic.

Use the files in this pack as the canonical spec.
